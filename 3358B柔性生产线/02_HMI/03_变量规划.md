# 变量规划

## 1. 分组原则

HMI 变量建议按四层分组：

- 整线公共变量
- 单站状态变量
- 单站手动变量
- 诊断/报警变量

## 2. 整线公共变量

建议建立 `HMI_Line` 变量组：

- `Line.CmdAuto`
- `Line.CmdStart`
- `Line.CmdStop`
- `Line.CmdReset`
- `Line.StsReady`
- `Line.StsRun`
- `Line.StsFault`
- `Line.CurrentStation`
- `Line.CurrentCycle`
- `Line.TotalCount`

## 3. 单站变量命名

建议统一格式：

```text
S01.Sts.Step
S01.Sts.Ready
S01.Sts.Busy
S01.Sts.Done
S01.Sts.Fault
S01.Cmd.ManTopValve
S01.Cmd.ManPushValve
S01.IO.InTopFwd
S01.IO.OutTopValve
```

5 个站统一为：

- `S01`
- `S02`
- `S03`
- `S04`
- `S05`

## 4. 变量来源建议

### 来自 DB_Station

- 运行状态
- 当前步号
- 自动/手动模式
- 输入输出状态
- 手动命令位

### 来自 DB_Comm

- `CmdAuto`
- `CmdStart`
- `CmdReset`
- `StsReady`
- `StsBusy`
- `StsDone`
- `StsFault`
- `StepNo`
- `HeartbeatTx`
- `TransferJob`
- `ReadyForJob`
- `JobDone`

### 来自 DB_Alarm

- `Active`
- `Material`
- `Sensor`
- `Actuator`
- `Motion`
- `AlarmCode`

### 来自 DB_Parm

- 时间参数
- 模拟量速度参数
- 延时参数
- 站号

## 6. HMI变量使用规则

- HMI 只写命令位和参数位，尽量不直接写状态位
- 手动页按钮要写 `Cmd/Man` 类变量，不要直接写 `Q`
- 所有 `IO` 变量优先显示只读状态
- 参数修改后要有“生效/保存”提示
