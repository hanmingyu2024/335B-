# 3358B公共命名规范总则

## 1. 目的

本规范用于统一 `3358B` 五站柔性生产线的命名方式，避免出现：

- 同一类对象多种写法
- 中英文混用无规则
- 变量名无法一眼看出所属站和用途
- HMI、PLC、通信、报警之间命名脱节

## 2. 使用范围

本规范适用于：

- 项目目录
- PLC 站名
- 程序块
- 数据块
- UDT
- 变量
- I/O 映射
- 通信点
- 报警编号
- HMI 画面和对象

## 4. 当前项目采用的统一策略

结合你现在已经生成的结构，建议固定为：

- 目录和站名：中文 + 站号
- 块文件名：ASCII 英文缩写
- PLC 变量字段：英文语义名
- HMI 画面对象：ASCII 前缀命名
- 报警文本：中文
- 报警编号：数字编码

## 5. 一句话原则

命名必须同时满足这 4 点：

1. 一眼看出对象类型
2. 一眼看出所属站
3. 一眼看出功能
4. 同类对象命名结构一致
