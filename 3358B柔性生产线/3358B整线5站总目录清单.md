# 3358B整线5站总目录清单

## 1. 整线站别划分

```text
01站 供料站
02站 加工站
03站 输送站
04站 装配站
05站 分拣站
```

## 2. 整线目录结构

```text

├─ 00_项目说明
├─ 01_设备和网络
│  ├─ 01站供料站 [CPU 1214C AC_DC_Rly]
```

```text
程序块
├─ 00_OB

├─ 30_通信
├─ 40_报警
```

## 4. 各站设备层差异

### 01站供料站

- 挡料气缸
- 推料气缸
- 供料输送
- 出料/到位传感器

### 02站加工站

- 夹紧气缸
- 加工执行气缸
- 加工电机或加工动作单元

### 03站输送站

- 主输送单元
- 搬运电机/运动轴
- 挡停气缸
- 中间工位传感器

### 04站装配站

- 定位/装配气缸
- 装配执行单元
- 装配到位传感器

### 05站分拣站

- 分拣推杆气缸
- 剔除气缸
- 出料输送
- OK/NG 检测传感器

## 5. 单站关键块

每站都建议保留：

- `Main [OB1]`
- `Startup [OB100]`
- `FC_IMap`
- `FC_QMap`
- `FB_StationCtrl`
- `FB_Comm`
- `FB_Alarm`
- `FB_Cylinder`
- `FB_Motor`
- `FB_Conveyor`
- `FB_Sensor`
- `DB_Station`
- `DB_Comm`
- `DB_Alarm`
- `DB_Parm`

## 6. 设计原则

- `OB1` 只调度，不堆业务逻辑
- 站控和设备层分开
- 通信、报警独立成层
- 实例 DB 跟随 FB 自动生成
- 参数用全局 `DB_Parm` 管理
- 5 个站尽量保持同构

## 7. 备注

- `OB123` 可保留，但不要替代 `OB100`
- 目录结构已经按五站模板建立，后续重点是补接口、补异常处理和细化 HMI
