# DB块填写规则

## 1. 先区分两类 DB

### 类型 A：实例 DB

对应某个 `FB` 的调用实例，例如：

- `DB_Station`
- `DB_Comm`
- `DB_Alarm`
- `DB_Cyl_Stopper`
- `DB_Cyl_Pusher_1`

### 类型 B：全局 DB

手工建立的数据块，例如：

- `DB_Parm`
- 后续可能的 `DB_HMI`
- `DB_Count`

## 2. 实例 DB 怎么处理

### 原则

不要先在实例 DB 里敲变量。

### 正确方法

1. 先打开对应 `FB`
2. 补 `Input / Output / InOut / Static`
3. 编译
4. 实例 `DB` 自动带出变量

## 3. 全局 DB 怎么处理

全局 DB 才需要你在表格里手工填写变量：

- 名称
- 数据类型
- 起始值
- 注释

## 4. DB_Parm 推荐写法

例如：

| 名称 | 数据类型 | 起始值 | 注释 |
|---|---|---|---|
| `StationNo` | `Int` | `1` | 工位号 |
| `tTopExtend` | `Time` | `T#800ms` | 顶料前进超时 |
| `tPushExtend` | `Time` | `T#800ms` | 推料前进超时 |
| `tAlarmDelay` | `Time` | `T#2s` | 报警延时 |

## 5. 你当前项目的建议

### 应该自动生成的

- `DB_Station`
- `DB_Comm`
- `DB_Alarm`
- 所有设备实例 DB

### 应该手填的

- `DB_Parm`

## 6. 结论

DB 页面里是否要手填，先问自己一句：

**这个 DB 是不是某个 FB 的实例？**

如果是：

- 不手填

如果不是：

- 手填变量格式
