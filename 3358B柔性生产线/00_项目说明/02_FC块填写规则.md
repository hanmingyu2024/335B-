# FC块填写规则

## 1. FC 的原则

`FC` 没有实例记忆，不带实例 DB。  
所以 FC 最适合：

- 纯逻辑处理
- 数据搬运
- 规则判断
- 不需要保存状态的子功能

## 2. 当前项目中的 FC

你现在主要有：

- `FC_IMap`
- `FC_QMap`
- `FC_ModeCtrl`
- `FC_AutoSeq`
- `FC_ManualCtrl`
- `FC_Reset`
- `FC_Lights`

## 3. 两种做法

### 做法 A：不传形参，直接用全局 DB

优点：

- 快
- 好上手
- 和你现在已有的 `SCL` 主体一致

缺点：

- 复用性一般
- 依赖固定 DB 名称

### 做法 B：用 InOut 传结构体

优点：

- 更规范
- 后面换站方便
- 更像正式工程

缺点：

- 写代码时要把 `DB_Station` 改成 `#stStation`

## 4. 当前最推荐方案

如果你现在以“先跑通项目”为主，建议：

### `FC_IMap`

- `InOut`
  - `stStation : "UDT_StationState"`

### `FC_QMap`

- `InOut`
  - `stStation : "UDT_StationState"`

### `FC_ModeCtrl`

- `InOut`
  - `stStation : "UDT_StationState"`

### `FC_ManualCtrl`

- `InOut`
  - `stStation : "UDT_StationState"`

### `FC_Reset`

- `InOut`
  - `stStation : "UDT_StationState"`

### `FC_Lights`

- `InOut`
  - `stStation : "UDT_StationState"`

### `FC_AutoSeq`

- `Input`
  - `stParm : "UDT_Parm"`
- `InOut`
  - `stStation : "UDT_StationState"`

## 5. Temp 建议

FC 里只有在你确实需要局部变量时才加 `Temp`。

例如：

- `tmpReady : Bool`
- `tmpFault : Bool`
- `tmpStepNext : Int`

否则先不要乱加。

## 6. 结论

对于 FC：

- 最关键的是 `InOut stStation`
- `FC_AutoSeq` 再额外加一个 `Input stParm`
