# 3358B 整线总控与握手策略

## 1. 总体架构

- `01站` 作为整线主控站，负责节拍、任务分发、异常停线与恢复。
- `03站` 作为搬运核心站，执行跨站取放动作（爪子水平轴 + 抓取执行机构）。
- `05站` 作为分拣末站，执行运输带轴速度控制与分拣执行。
- 其他站保留本地工艺自治，但由整线命令统一启动/停止/复位。

## 2. 通讯建议（站间）

- 首选拓扑：PROFINET 工业以太网。
- 数据模型：每站统一 `DB_Comm`，包含命令区和状态区。
- 映射方式（二选一，项目统一）：
  - 方案A：S7 通信 `PUT/GET`（实施快，适合现有项目改造）。
  - 方案B：I-Device / 共享IO映射（实时性更强，前期配置更重）。
- 心跳建议：每站 `HeartbeatTx` 500ms 翻转一次，主控侧按超时判离线。

## 3. 主控下发与站点上报

主控下发：

- `CmdAuto`
- `CmdStart`
- `CmdStop`
- `CmdReset`

站点上报：

- `StsReady`
- `StsBusy`
- `StsDone`
- `StsFault`
- `StepNo`

## 4. 标准交接握手（相邻站）

统一使用 8 信号闭环：

- `ReqOut`
- `PartReady`
- `AllowTake`
- `TakeDone`
- `ReqIn`
- `AllowIn`
- `PlaceDone`
- `PartReceived`

建议时序：

1. 源站置位 `ReqOut + PartReady`
2. 搬运/目标站确认 `AllowTake`
3. 搬运完成后置位 `TakeDone`
4. 目标站准备好后置位 `AllowIn`
5. 放料完成后置位 `PlaceDone`
6. 目标站确认 `PartReceived`
7. 双方清信号，回待机

## 5. 03站（爪子水平轴）策略

轴相关 I/O 基线（必须存在）：

- 输入：`I0.0` 原点，`I0.1` 右限位，`I0.2` 左限位
- 输出：`Q0.0` 脉冲，`Q0.1` 方向

控制建议：

- `03站` 对外暴露任务接口：`TransferJob/JobSource/JobTarget`。
- 取放动作前必须完成原点与限位互锁校验。
- 回零速度限制建议：低速（示教阶段 <= 25）。
- 单次任务结束信号：`JobDone`；执行中：`JobBusy`；可接新任务：`ReadyForJob`。

## 6. 05站（运输带轴）策略

轴相关 I/O 基线（必须存在）：

- 编码器输入：`I0.0` B相，`I0.1` A相，`I0.2` Z相
- 执行输出：`Q0.0` 电机启动，`QW80` 模拟量速度给定

控制建议：

- 速度闭环由站内完成，整线仅给分拣节拍窗口与运行许可。
- `QW80` 建议做限幅与斜坡，避免上电突变。
- 分拣动作与上游交接完成信号联锁，防止误分拣。

## 7. 异常与互锁

- 任一站 `StsFault=1`，主控应触发整线停机流程。
- 站间握手超时要报警并冻结当前链路，避免重复取放。
- `CmdReset` 需清故障 + 清交接中间位 + 复位步号。

## 8. 联调顺序

1. 单站调通 `CmdAuto/CmdStart/CmdStop/CmdReset`。
2. 联调每条站间握手链路（先不用真实负载）。
3. 调 03站 轴回零与定位，再接入任务分配。
4. 调 05站 编码器计数与 `QW80` 速度控制。
5. 最后联调整线节拍与故障恢复流程。
