# 画面结构与导航

## 1. 画面层级

建议采用三层结构：

1. 一级：整线入口页
2. 二级：整线运行与公共功能页
3. 三级：各站详细页

## 2. 推荐画面树

```text
HMI
├─ 00_启动页
├─ 01_整线首页
├─ 02_整线自动总览
├─ 03_整线工艺流向
├─ 10_01供料站监控
├─ 11_01供料站手动
├─ 20_02加工站监控
├─ 21_02加工站手动
├─ 30_03输送站监控
├─ 31_03输送站手动
├─ 40_04装配站监控
├─ 41_04装配站手动
├─ 50_05分拣站监控
├─ 51_05分拣站手动
├─ 60_参数设置
├─ 70_报警页
├─ 80_IO诊断
├─ 81_通信诊断
├─ 82_用户登录
└─ 90_统计与教学演示
```

## 3. 固定导航区

所有页面建议固定保留顶部或左侧导航区，至少包含：

- 首页
- 整线总览
- 当前站监控
- 当前站手动
- 报警
- 参数
- 诊断
- 登录/退出

## 4. 首页布局建议

`01_整线首页` 建议放：

- 五站工艺示意图
- 每站三色状态块：待机/运行/故障
- 整线启动、停止、复位按钮
- 自动/手动模式指示
- 当前故障站
- 当前整线节拍或循环计数

## 5. 单站监控页布局建议

每个站的监控页统一布局：

- 左上：站名、模式、当前步号
- 左中：工艺流程简图
- 右上：输入传感器状态
- 右中：输出动作状态
- 下方：本站报警、完成信号、与上下游握手

## 6. 单站手动页布局建议

每个站的手动页统一布局：

- 左侧：手动互锁状态
- 中间：动作按钮区
- 右侧：到位反馈区
- 底部：动作说明和注意事项

手动页必须区分：

- 点动动作
- 保持动作
- 原点回归
- 禁止在自动状态下操作

## 7. 报警与诊断入口

报警页和诊断页不能藏太深，建议：

- 顶部固定报警按钮
- 当前报警数常显
- 站故障时点击站块直接跳转到该站监控页
