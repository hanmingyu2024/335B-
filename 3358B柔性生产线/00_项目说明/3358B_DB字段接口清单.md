# 3358B DB字段接口清单

## 核心关系
- `DB_Station`：站控主数据块，供 `FC_IMap/FC_QMap/FC_AutoSeq/FC_ManualCtrl/FB_StationCtrl` 共用。
- `DB_Comm`：站间和主控通信数据块。
- `DB_Alarm`：报警汇总数据块。
- `DB_Parm`：本站参数块。
- `UDT_StationState / UDT_CommCmd / UDT_CommSts / UDT_AlarmItem / UDT_Parm`：统一数据类型参考模板。

## DB_Station重点字段
- 模式命令：`PB_Start`、`PB_Stop`、`PB_EStop`、`SelSingle`、`ModeAuto`、`ModeManual`、`RunCmd`
- 站状态：`StationReady`、`StationBusy`、`StationDone`、`CycleActive`、`Step`
- 故障：`FaultActive`、`FaultMaterial`、`FaultSensor`、`FaultActuator`、`FaultMotion`
- 指示灯：`LampCmdYellow`、`LampCmdGreen`、`LampCmdRed`
- 现场输入：按各站 `FC_IMap.scl` 映射到 `In...` 字段
- 现场输出：按各站 `FC_QMap.scl` 从 `Out...` 字段映射到 `Q`
- 手动命令：`Man...` 字段供 HMI 或调试按钮使用

## DB_Comm重点字段
- 主控命令：`CmdAuto`、`CmdStart`、`CmdStop`、`CmdReset`
- 状态返回：`StsReady`、`StsBusy`、`StsDone`、`StsFault`、`StepNo`
- 交接握手：`ReqOut`、`PartReady`、`AllowTake`、`TakeDone`、`ReqIn`、`AllowIn`、`PlaceDone`、`PartReceived`、`ProcessDone`、`DoneAck`
- 搬运站专用：`TransferJob`、`ReadyForJob`、`JobBusy`、`JobDone`、`JobAbort`、`JobSource`、`JobTarget`、`PickAtLeft`、`PlaceAtLeft`、`PlaceRotateLeft`

## DB_Line重点字段
- 整线命令：`CmdAuto`、`CmdStart`、`CmdStop`、`CmdReset`
- 整线状态：`StsReady`、`StsRun`、`StsFault`、`StsPaused`、`CycleActive`
- 节拍与流程：`Step`、`CurrentCycle`、`TotalCount`
- 当前任务：`ActiveSource`、`ActiveTarget`
- 故障汇总：`FaultCode`

## DB_Alarm重点字段
- `Active`
- `Material`
- `Sensor`
- `Actuator`
- `Motion`
- `AlarmCode`

## 使用说明
- 现在生成的是统一模板，字段取并集，不是每个站都必须全部使用。
- 导入博途时，先按这些 `.db/.udt` 文件建数据结构，再把前面已经生成的 `.scl` 主体代码贴入对应块。
- 如果你后面决定把 `DB_Station` 改成 UDT 嵌套结构，需要同步修改现有 `SCL` 中的字段访问路径。
