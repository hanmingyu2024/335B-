# 3358B 站间通信点表（代码抽取版）

## 1. 统一命令（主控 -> 各站）

| 信号 | 说明 |
|---|---|
| `CmdAuto` | 自动模式命令 |
| `CmdStart` | 启动命令 |
| `CmdStop` | 停止命令 |
| `CmdReset` | 故障复位命令 |

## 2. 统一状态（各站 -> 主控）

| 信号 | 说明 |
|---|---|
| `StsReady` | 就绪 |
| `StsBusy` | 运行中 |
| `StsDone` | 当前周期完成 |
| `StsFault` | 故障 |
| `StepNo` | 当前步号 |
| `HeartbeatTx` | 本站发出的心跳位 |

## 3. 标准交接握手（标准 8 位）

| 信号 | 含义 |
|---|---|
| `ReqOut` | 源站请求出站 |
| `PartReady` | 源站出料位有件 |
| `AllowTake` | 允许取件 |
| `TakeDone` | 取件完成 |
| `ReqIn` | 请求入站 |
| `AllowIn` | 允许入站 |
| `PlaceDone` | 放件完成 |
| `PartReceived` | 目标站收件确认 |

## 4. 代码实锤的方向规则

- `01站` 的 `DB_Link_S02..S05` 是远端站 `DB_Comm` 的镜像影子块，方向必须按“谁赋值谁是源”来配。
- 心跳不是同名直连，而是交叉映射：
  - `01.DB_Link_Sxx.HeartbeatTx -> 远端.DB_Comm.HeartbeatRx`
  - `远端.DB_Comm.HeartbeatTx -> 01.DB_Link_Sxx.HeartbeatRx`
- 下列字段只在本地维护，不做站间 PUT/GET：`HeartbeatRxLast`、`HeartbeatWatchdog`、`HeartbeatLimit`、`PeerAlive`。
- 本节依据的主要代码块：
  - `01站`：`70_整线/FB_LineCtrl.scl`
  - `02站`：`30_通信/FB_Comm.scl`、`20_站控/FC_AutoSeq.scl`
  - `03站`：`30_通信/FB_Comm.scl`、`20_站控/FC_AutoSeq.scl`
  - `04站`：`30_通信/FB_Comm.scl`、`20_站控/FC_AutoSeq.scl`
  - `05站`：`30_通信/FB_Comm.scl`、`20_站控/FC_AutoSeq.scl`

| 链路 | `01 -> 站` 建议 PUT | `站 -> 01` 建议 GET | 可选/保留 |

| `S02` | `Cmd*`、`HeartbeatTx -> HeartbeatRx`、`PlaceDone`、`TakeDone` | `HeartbeatTx -> HeartbeatRx`、`Sts*`、`AllowIn`、`PartReceived`、`ProcessDone`、`ReqOut`、`PartReady` | `ReqIn`、`AllowTake` |
| `S03` | `Cmd*`、`HeartbeatTx -> HeartbeatRx`、`TransferJob`、`JobSource`、`JobTarget`、`PlaceRotateLeft` | `HeartbeatTx -> HeartbeatRx`、`Sts*`、`ReadyForJob`、`JobBusy`、`JobDone` | `JobAbort`、`PickAtLeft`、`PlaceAtLeft` |

## 7. 调试检查项

1. 心跳是否按交叉关系配置，而不是 `Tx -> Tx` / `Rx -> Rx` 同名直连。
2. 只映射真正参与逻辑的字段，避免把 `PeerAlive`、`HeartbeatWatchdog` 这类本地诊断位错误上网。
3. 对 `S03` 任务接口只先配最小集：`TransferJob/JobSource/JobTarget/PlaceRotateLeft`。
4. 对 `S02/S04/S05` 先把 `PlaceDone/TakeDone/DoneAck` 这些闭环位跑通，再决定是否保留 `ReqIn/AllowTake` 作为可视化辅助位。
