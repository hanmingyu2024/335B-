# 示例与反例

## 1. 好的命名

### 块名

- `FB_StationCtrl`
- `FC_AutoSeq`
- `DB_Cyl_Pusher_1`
- `UDT_CommCmd`

### 变量名

- `DB_Station.InTopFwd`
- `DB_Station.OutPushValve`
- `DB_Station.ManGripClamp`
- `DB_Comm.StsReady`
- `S03.Comm.TransferJob`

### HMI对象

- `Btn_S01_ManPush`
- `Lamp_S04_Fault`
- `Num_S05_SortClass`

## 2. 不好的命名

### 块名反例

- `自动程序1`
- `新块`
- `FB1_Test`
- `通信块改`

### 变量名反例

- `a`
- `flag1`
- `M_Run`
- `Top`
- `Output1`

### HMI对象反例

- `按钮1`
- `新建对象`
- `灯`
- `数字显示3`

## 3. 对比示例

| 不推荐 | 推荐 |
|---|---|
| `Top1` | `InTopFwd` |
| `PushOut` | `OutPushValve` |
| `StartFlag` | `RunCmd` |
| `Alarm1` | `FaultMaterial` |
| `Page1` | `Scr_S01_Monitor` |
| `button_start` | `Btn_LineStart` |

## 4. 执行建议

- 新建对象时先按规范命名，再开始写逻辑
- 发现旧命名混乱时优先在当前站内一次性统一
- 不要边开发边切换命名风格
