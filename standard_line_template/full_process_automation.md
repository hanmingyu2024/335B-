# 全流程自动化说明

这份模板把你图片里的七个阶段固定成一个可执行流水线。

## 1. IO 对点

输入：

- 站点清单
- CPU 型号
- I/O 数量
- I/O 命名规则

输出：

- `04_IOList/io_summary.md`
- 各站 I/O 规模占位信息
- 待补充的 `FC_IMap` / `FC_QMap`

Openness 对应能力：

- 变量与符号管理
- PLC Tag 导入导出
- 分布式 I/O 组态

## 2. 工艺流程

输入：

- 站间流向
- 节拍逻辑
- 交接握手字段

输出：

- `00_ProjectInfo/02_process_flow.md`
- `03_CommTable/station_comm_signals.md`
- `FC_AutoSeq` 骨架
- `FB_LineCtrl` 骨架

Openness 对应能力：

- 程序块创建
- 程序结构管理
- 连接与通讯组态

## 3. 设备组态

输入：

- 每站 CPU 订货号
- 设备名
- DeviceItemName
- 后续扩展模块规划

输出：

- 产线清单中的 `cpuTypeIdentifier`
- TIA 设备自动创建脚本

Openness 对应能力：

- 创建设备
- 配置网络
- 配置设备与模块

## 4. 项目框架搭建

输入：

- 标准目录规范
- 标准块清单
- 标准数据块与 UDT

输出：

- 标准工作区目录
- `tia_sources/StationXX`
- `manifest.json`

Openness 对应能力：

- 项目生命周期管理
- 项目结构操作
- 外部源文件导入

## 5. 编程

输入：

- 设备模板
- 通讯模板
- 报警模板

输出：

- `FB_StationCtrl`
- `FB_Comm`
- `FB_Alarm`
- `FB_Cylinder`
- `FB_Motor`
- `FB_Conveyor`
- `FB_Sensor`

说明：

这一步当前是“标准骨架自动生成 + 专有逻辑人工补齐”。

## 6. 调试

输入：

- 编译日志
- 现场 PLC 连接信息

输出：

- `logs/*_compile.txt`
- 编译错误定位

Openness 对应能力：

- 单设备编译
- 编译日志提取
- 下载 / 上传
- 在线诊断

说明：

当前仓库已经实现“自动编译与归档”，现场下载和在线调试还保留为下一步扩展。

## 7. 优化

输入：

- 编译问题
- 节拍问题
- I/O 地址变更
- 命名和结构审查结果

输出：

- 清单修订
- 脚本重跑
- 新版 `.ap17/.zap17`

核心原则：

- 所有工程化差异尽量回到清单
- 所有可复用结构尽量回到模板
- 所有交付物尽量通过脚本重建

## 建议的下一个项目工作法

1. 先写清单，不先写块。
2. 先跑脚手架，不先手建目录。
3. 先生成工程骨架并编译，不先堆完整逻辑。
4. 把专有工艺限制在 `FC_AutoSeq` 和少量设备实例层。
5. 修改一律回写到清单或模板，不直接在某个项目里“临时修”。
