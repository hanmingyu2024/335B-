# 3358B 博途块接口填写说明

## 1. 先说结论

你现在这个项目里，`OB/FB/FC/DB` 这 4 类块不要按同一种方式处理。

### 正确理解

- `OB块`：一般**不需要你自定义形参**
- `FC块`：可以有形参，**推荐通过 InOut 传结构体**
- `FB块`：必须设计接口，**实例 DB 的内容来自 FB 接口和 Static**
- `DB块`：要先分清楚是**全局 DB**还是**实例 DB**

## 2. 最关键的区别

### 2.1 实例 DB

例如：

- `DB_Station [DB1]`
- `DB_Comm [DB2]`
- `DB_Alarm [DB4]`

如果它们是对应：

- `FB_StationCtrl`
- `FB_Comm`
- `FB_Alarm`

的**实例 DB**，那么：

**不要在 DB 里手工先填变量。**

正确顺序是：

1. 先把 `FB` 的 `Input / Output / InOut / Static` 填好
2. 再创建或更新实例 `DB`
3. `DB` 会自动带出变量结构

也就是说：

**实例 DB 的变量格式，来源于 FB，不来源于你在 DB 页面里手敲。**

### 2.2 全局 DB

例如：

- `DB_Parm`
- 以后如果你单独建 `DB_HMI`
- `DB_Recipe`

这种**全局 DB**，才是你自己在 `Static` 区手工填写变量名、数据类型、初值。

## 3. 当前项目建议

结合你现在的结构，建议这样分：

### 不手填变量的 DB

- `DB_Station`
- `DB_Comm`
- `DB_Alarm`
- 各设备实例 DB，如：
  - `DB_Cyl_Stopper`
  - `DB_Cyl_Pusher_1`
  - `DB_Conv_Feed_1`

这些都建议作为 **FB 的实例 DB**

### 需要手填变量的 DB

- `DB_Parm`
- 后续如果有：
  - `DB_HMI`
  - `DB_Recipe`
  - `DB_Count`
  - `DB_Report`

## 4. 你现在最容易出错的点

你截图里 `DB_Station [DB1]` 页面是空的，不代表 DB 坏了，而是因为：

- `FB_StationCtrl` 还没把接口和 `Static` 定义完整

所以 `DB_Station` 没东西可以映射出来。

一句话：

**先补 FB 接口，再看实例 DB。**
