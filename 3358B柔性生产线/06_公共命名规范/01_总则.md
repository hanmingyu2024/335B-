# 总则

## 1. 基本规则

- 文件名、块名、变量名优先使用 ASCII
- 同类对象统一前缀
- 不使用空格
- 单词首字母大写，前缀保持固定
- 站别必须能从命名里看出来
- 不使用无含义缩写，如 `Temp1`、`FlagA`

## 2. 推荐命名结构

```text
前缀_对象类型_功能_序号
```

例如：

```text
DB_Cyl_Pusher_1
FB_StationCtrl
UDT_CommCmd
Scr_S01_Monitor
```

## 3. 站别代码

| 站别 | 中文名称 | 代码 |
|---|---|---|
| 01 | 供料站 | `S01` |
| 02 | 加工站 | `S02` |
| 03 | 输送站 | `S03` |
| 04 | 装配站 | `S04` |
| 05 | 分拣站 | `S05` |

## 4. 类型前缀

| 类型 | 前缀 |
|---|---|
| 组织块 | `OB` |
| 函数 | `FC` |
| 功能块 | `FB` |
| 数据块 | `DB` |
| 数据类型 | `UDT` |
| 页面 | `Scr` |
| 按钮 | `Btn` |
| 指示灯 | `Lamp` |
| 文本 | `Txt` |
| 数值 | `Num` |
| 趋势 | `Trend` |

## 5. 禁止项

- 禁止同一类对象有多种前缀
- 禁止中文和拼音随意混写在块名中
- 禁止 `M1`、`Data1`、`test` 这类无语义名字
- 禁止同一功能今天叫 `Push` 明天叫 `Pusher`
