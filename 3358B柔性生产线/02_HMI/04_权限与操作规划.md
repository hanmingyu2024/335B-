# 权限与操作规划

## 1. 角色划分

建议最少分 4 级：

| 角色 | 说明 | 典型权限 |
|---|---|---|
| 游客 | 只看不操作 | 查看状态、查看报警 |
| 操作员 | 日常运行 | 启动、停止、复位、报警确认 |
| 调试员 | 单站调试 | 手动动作、单步测试、IO诊断 |
| 工程师 | 高级维护 | 参数修改、权限设置、通信调试 |

## 2. 权限边界

### 游客

- 可看首页、总览、单站监控、报警
- 不可启动、不可手动、不可改参数

### 操作员

- 可启动、停止、复位
- 可报警确认
- 不可进入危险手动页

### 调试员

- 可进入单站手动页
- 可执行点动动作
- 可进入 IO 诊断页
- 不可修改系统级通信参数

### 工程师

- 可修改时间参数
- 可修改模拟量速度参数
- 可查看通信诊断和高级诊断
- 可维护用户权限

## 3. 操作互锁要求

- 自动模式下禁止进入手动动作输出
- 手动模式下必须停止自动顺控
- 急停有效时所有手动按钮禁用
- 故障未复位时禁止再次启动
- 关键动作建议二次确认

## 4. 页面权限建议

| 页面 | 游客 | 操作员 | 调试员 | 工程师 |
|---|---:|---:|---:|---:|
| 整线首页 | 是 | 是 | 是 | 是 |
| 整线总览 | 是 | 是 | 是 | 是 |
| 单站监控 | 是 | 是 | 是 | 是 |
| 单站手动 | 否 | 否 | 是 | 是 |
| 参数设置 | 否 | 否 | 否 | 是 |
| 报警页 | 是 | 是 | 是 | 是 |
| IO诊断 | 否 | 否 | 是 | 是 |
| 通信诊断 | 否 | 否 | 否 | 是 |

## 5. 按钮行为规范

- 启动：写入 `CmdStart`
- 停止：写入 `CmdStop`
- 复位：写入 `CmdReset`
- 手动动作：写入 `Man...`
- 参数写入：先缓存，再确认下发

## 6. 实训线建议

这类柔性产线通常兼顾教学，建议保留“演示模式”：

- 屏蔽高风险动作
- 允许只看流程不动作
- 可以单步演示站间传递逻辑
