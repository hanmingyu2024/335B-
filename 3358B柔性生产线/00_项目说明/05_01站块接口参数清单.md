## 3. FB块

### FB_StationCtrl

建议这样填：

| 区域 | 名称 | 数据类型 | 说明 |
|---|---|---|---|
| Input | `stParm` | `"UDT_Parm"` | 工位参数 |
| InOut | `stStation` | `"UDT_StationState"` | 工位状态主结构 |
| Static | `tmStep` | `TON` | 步序定时器 |
| Static | `tmAlarm` | `TON` | 报警超时定时器 |
| Static | `lastStep` | `Int` | 上一扫描步号 |

### FB_Comm

| 区域 | 名称 | 数据类型 | 说明 |
|---|---|---|---|
| Input | `CmdAuto` | `Bool` | 自动命令 |
| Input | `CmdStart` | `Bool` | 启动命令 |
| Input | `CmdReset` | `Bool` | 复位命令 |
| Input | `HeartbeatRx` | `Bool` | 上位/主站心跳 |
| Output | `HeartbeatTx` | `Bool` | 本站心跳 |
| Output | `StsReady` | `Bool` | 就绪 |
| Output | `StsBusy` | `Bool` | 运行中 |
| Output | `StsDone` | `Bool` | 完成 |

| Output | `StepNo` | `Int` | 当前步号 |
| InOut | `stStation` | `"UDT_StationState"` | 工位状态结构 |
| Static | `lastHb` | `Bool` | 心跳翻转记忆 |

## 4. DB块

### DB_Station

- 类型：`FB_StationCtrl` 的实例 DB
- 处理方式：**不要手工填写变量**
- 先补 `FB_StationCtrl` 接口和 `Static`

### DB_Comm

- 类型：`FB_Comm` 的实例 DB
- 处理方式：**不要手工填写变量**

### DB_Alarm

- 类型：`FB_Alarm` 的实例 DB
- 处理方式：**不要手工填写变量**

### DB_Parm

这个是全局 DB，需要手工填写变量。  
推荐最少填写：

| 名称 | 数据类型 | 起始值 | 说明 |
|---|---|---|---|
| `StationNo` | `Int` | `1` | 工位号 |
| `tTopExtend` | `Time` | `T#800ms` | 顶料前进超时 |
| `tPushExtend` | `Time` | `T#800ms` | 推料前进超时 |
| `tOutPosWait` | `Time` | `T#1s` | 出料位等待 |
| `tAlarmDelay` | `Time` | `T#2s` | 报警延时 |
